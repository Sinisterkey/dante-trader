# Enhanced Multi-Agent Trading System Plan

## Overview
Build a comprehensive modular algorithmic trading system for NAS100 trading with broker API integration, position management, trade logging, performance analytics, risk engine, machine learning integration, and advanced GUI components.

## Core Components

### 1. MarketIntelligence Class (Core Strategy)
- Translates Pine Script logic to Python
- Instrument: NAS100 (configurable)
- Lookback: 50 candles for Swing High/Low
- Indicators: SMA(50) for trend, ATR(14) for volatility
- Session Filter: London/NY Overlap (12:00 - 16:00 GMT)
- Confirmation: M15 signals align with 4H trend
- Risk Management: SL = 1.5 * ATR, TP = 2.5 * Risk, Risk = 1.5% of equity

### 2. Multi-Agent Architecture (CrewAI)
- Agent A: Technical Observer
  - Processes raw MT5 data
  - Identifies breakouts/pullbacks based on SMA50 and Swing levels
- Agent B: Executive Strategist
  - Validates 4H trend, session timing
  - Calculates final trade parameters
  - Provides detailed reasoning

### 3. Broker API Integration Layer
- Connectivity with brokerage APIs (MetaTrader 5, Interactive Brokers)
- Real-time data streaming for market prices
- Order execution capabilities (market, limit, stop orders)
- Account information retrieval (balance, margin, positions)
- Error handling and reconnection logic

### 4. Position Manager
- Automated trade execution based on agent signals
- Precise Stop Loss and Take Profit order placement and management
- Position sizing calculations based on risk parameters
- Order modification and cancellation capabilities
- Real-time position monitoring and P&L tracking

### 5. Trade Logger
- Comprehensive database system for recording all executed trades
- Captures entry/exit timestamps, prices, quantities
- Records realized/unrealized P&L, trade duration, slippage
- Strategy identification and tagging for performance analysis
- Audit trail for compliance and analysis

### 6. Performance Analytics Module
- Calculates key performance metrics:
  * Win rate (percentage of profitable trades)
  * Average profit per trade
  * Profit factor (gross profit/gross loss)
  * Maximum drawdown
  * Sharpe ratio
  * Expectancy
- Trade analysis by time of day, day of week, market conditions
- Equity curve visualization
- Rolling performance metrics

### 7. Risk Engine
- Dynamic position-sizing based on real-time account equity
- Volatility-adjusted position sizing
- Correlation-based risk management
- Maximum drawdown protection
- Daily loss limits
- Position concentration limits

### 8. Machine Learning Integration
- Feedback loop for strategy optimization
- Feature engineering from market data and trade outcomes
- Model training on historical trade performance
- Online learning for adapting to changing market conditions
- Strategy parameter optimization
- Signal confidence adjustment based on historical accuracy

### 9. Enhanced GUI Components
- Agent Thought Process Monitor: Real-time stream of agent internal reasoning
- Interactive Charting Window: High-fidelity candlestick charting with adjustable timeframes
- Integrated dashboard combining all system components

### 4. Technical Requirements & File Structure
- Language: Python 3.10+
- Libraries: MetaTrader5, pandas, pandas_ta, crewai, streamlit, python-telegram-bot, scikit-learn, ta-lib, plotly, sqlalchemy
- Modular Files:
  - config.py: Settings and constants (trading system configuration)
  - broker_integration.py: Broker API connectivity (MT5, IBKR, etc.)
  - position_manager.py: Trade execution and position management logic
  - trade_logger.py: Database system for recording trades
  - performance_analytics.py: Performance metrics calculation and reporting
  - risk_engine.py: Dynamic position sizing and risk management
  - ml_integration.py: Machine learning feedback loop and optimization
  - market_intelligence.py: Core trading strategy (enhanced from bridge.py)
  - agents.py: CrewAI agent definitions and logic
  - dashboard.py: Enhanced GUI with Agent Thought Process Monitor and Interactive Charting
  - app.py: Main application orchestrator

## Key Technical Specifications

### MarketIntelligence Class Methods:
- `get_swing_levels(data, lookback=50)`: Calculate swing high/low (excluding current candle)
- `calculate_sma(data, period=50)`: SMA50 for trend analysis
- `calculate_ema(data, period=20)`: EMA20 for short-term trend
- `calculate_rsi(data, period=14)`: RSI for momentum measurement
- `calculate_macd(data, fast=12, slow=26, signal=9)`: MACD for trend changes
- `calculate_bollinger_bands(data, period=20, std=2)`: Bollinger Bands for volatility
- `calculate_atr(data, period=14)`: ATR14 for volatility calculation (using True Range)
- `is_london_ny_overlap(timestamp)`: Check if time is within London/NY overlap (12:00-16:00 GMT)
- `is_asian_session(timestamp)`: Check if time is within Asian session (00:00-08:00 GMT)
- `align_timeframes(m15_data, h4_data)`: Confirm M15 signals align with H4 trend direction
- `generate_signal(data)`: Main signal generation logic combining technical indicators, swing levels, and session filter
- `calculate_signal_confidence(data, signal)`: Calculate confidence score for generated signal

### Performance Analytics Calculations:
- Win Rate = (Number of Winning Trades / Total Trades) × 100
- Average Profit = Total Profit / Number of Winning Trades
- Average Loss = Total Loss / Number of Losing Trades
- Profit Factor = Gross Profit / Gross Loss
- Expectancy = (Win Rate × Average Profit) - ((1 - Win Rate) × Average Loss)
- Maximum Drawdown = Peak-to-trough decline during a specific period
- Sharpe Ratio = (Average Return - Risk-Free Rate) / Return Standard Deviation
- Sortino Ratio = (Average Return - Target Return) / Downside Deviation
- Calmar Ratio = Annual Return / Maximum Drawdown
- Recovery Factor = Net Profit / Maximum Drawdown
- Payoff Ratio = Average Win / Average Loss

### Risk Engine Mechanisms:
- Fixed Fractional Position Sizing: Risk % of equity per trade
- Kelly Criterion Position Sizing: Optimal fraction based on win rate and payoff
- Volatility-Based Sizing: Position size inversely proportional to volatility
- Maximum Daily Loss Limit: Halt trading after daily loss threshold
- Maximum Drawdown Limit: Reduce position sizes after significant drawdown
- Correlation Limits: Limit exposure to highly correlated instruments
- Sector/Market Exposure Limits: Diversification controls
- Margin Utilization Limits: Prevent over-leveraging

### Risk Management Calculations:
- Stop Loss: Entry ± (1.5 * ATR) [configurable]
- Take Profit: Entry ± (2.5 * Risk) [configurable]
- Risk Amount: Equity * Risk_Percent [default 1.5%]
- Position Size: Risk Amount / (Stop Loss distance in points * Point Value)
- Leverage Adjustment: Position size accounting for available margin
- Margin Required: Position size * Contract Size * Entry Price / Leverage
