# Multi-Agent Trading System Plan

## Overview
Build a modular Multi-Agent Trading System for NAS100 trading with MT5 integration, CrewAI agents, Streamlit dashboard, and Telegram alerts.

## Core Components

### 1. MarketIntelligence Class (Core Strategy)
- Translates Pine Script logic to Python
- Instrument: NAS100
- Lookback: 50 candles for Swing High/Low
- Indicators: SMA(50) for trend, ATR(14) for volatility
- Session Filter: London/NY Overlap (12:00 - 16:00 GMT)
- Confirmation: M15 signals align with 4H trend
- Risk Management: SL = 1.5 * ATR, TP = 2.5 * Risk, Risk = 1.5% of equity

### 2. Multi-Agent Architecture (CrewAI)
- Agent A: Technical Observer
  - Processes raw MT5 data
  - Identifies breakouts/pullbacks based on SMA50 and Swing levels
- Agent B: Executive Strategist
  - Validates 4H trend, session timing
  - Calculates final trade parameters
  - Provides detailed reasoning

### 3. Visual Dashboard & Communication
- Streamlit UI showing live market state
- Live feed of agents' "Inner Monologue" (thought process)
- Telegram Bot alerts for valid trades with Action, Entry, SL, TP, Reasoning

### 4. Technical Requirements & File Structure
- Language: Python 3.10+
- Libraries: MetaTrader5, pandas, pandas_ta, crewai, streamlit, python-telegram-bot
- Modular Files:
  - config.py: Settings and constants (NAS100-specific)
  - bridge.py: MT5 data fetching and connection handling (includes MarketIntelligence)
  - agents.py: CrewAI agent definitions and logic
  - app.py: Streamlit UI and main application loop

## Key Technical Specifications

### MarketIntelligence Class Methods:
- `get_swing_levels(data, lookback=50)`: Calculate swing high/low (excluding current candle)
- `calculate_sma(data, period=50)`: SMA50 for trend analysis
- `calculate_atr(data, period=14)`: ATR14 for volatility calculation (using True Range)
- `is_london_ny_overlap(timestamp)`: Check if time is within London/NY overlap (12:00-16:00 GMT)
- `align_timeframes(m15_data, h4_data)`: Confirm M15 signals align with H4 trend direction
- `generate_signal(data)`: Main signal generation logic combining swing levels, SMAs, and session filter

### Error Handling & Logging:
- MT5 connection with automatic reconnection on failure
- Timeout handling for MT5 operations (60 second timeout)
- Graceful degradation when market data is unavailable
- Comprehensive logging of agent decisions and market analysis
- Exception handling in all external API calls (MT5, Telegram)

### Risk Management Calculations:
- Stop Loss: Entry ± (1.5 * ATR) 
- Take Profit: Entry ± (2.5 * Risk)
- Risk Amount: Equity * 0.015 (1.5%)
- Position Size: Risk Amount / (Stop Loss distance in points)

### CrewAI Agent Responsibilities:
**Technical Observer:**
- Input: Raw OHLCV data from bridge
- Processing: Pattern recognition (swings, SMA crosses)
- Output: Signal direction, strength, preliminary reasoning

**Executive Strategist:**
- Input: Signal from Technical Observer + higher timeframe data
- Processing: Trend validation, session check, risk calc
- Output: Final trade decision with detailed reasoning

### Streamlit Dashboard Components:
- Real-time price chart with indicators
- Session status indicator (green/red for London/NY overlap)
- Agent thought process scrolling feed
- Trade signal history table
- Performance metrics (win rate, avg profit, etc.)
- Connection status indicators (MT5, Telegram)
